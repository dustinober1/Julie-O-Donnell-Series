# Autonomous Systems Reference

## Purpose

Support future books involving autonomous targeting, defense logistics, drones, procurement, evaluation systems, or semi-autonomous decision tools.

## Use This File For

- Human authorization ambiguity.
- Evaluation gaming.
- Contractor testing incentives.
- Edge-case failures.
- Logistics optimization.
- Autonomous escalation pressure.
- Rules of engagement as system constraints.

## Possible Thriller Engine

A system passes evaluation because the test environment was gamed, but the deployed system fails dangerously in messy real-world conditions.

## Julie Connection

Autonomous systems can bring the series back toward Julie's original Argus Beta wound while still using a fresh technical mechanism.

## Guardrail

Keep chain of command, human authorization, and technical limits plausible. Do not turn autonomous systems into science-fiction robots unless the series intentionally shifts genre.
