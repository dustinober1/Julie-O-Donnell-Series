# Skill: Julie Series Memory

## Purpose

Maintain long-range memory for the Julie O'Donnell series across books, arcs, recurring characters, institutional consequences, unresolved threads, and reader expectations.

This skill is not the canon file itself. It defines how to use memory files correctly.

## Memory Sources

Use these files for durable memory:

- `.opencode/skills/julie-series-canon.md`
- `series-bible/00-series-dashboard.md`
- `series-bible/04-series-timeline.md`
- `series-bible/09-continuity-locks.md`
- `series-bible/10-book-by-book-series-arc.md`
- `series-bible/unresolved-series-threads.md`
- each book's `book-retrospective.md`
- each book's `continuity-log.md`
- each book's `chapter-facts.md`

## Memory Rules

- Do not trust only the current chapter.
- Do not rely on remembered assumptions when a file exists.
- Do not promote temporary details into global memory.
- Do not erase unresolved threads unless the manuscript resolves them.
- Record consequences from each book so future books do not reset the world.

## After Each Book

A book retrospective should capture:

- final title;
- final major case outcome;
- technical threat used;
- villain/motive revealed;
- institutional damage or cover-up;
- public/private consequence for Julie;
- relationship changes;
- unresolved threads carried forward;
- technical motifs not to repeat too soon;
- reader-facing promise for next book.

## Series Drift Risks

Watch for:

- Julie becoming too healed too quickly;
- the same AI exploit repeating under a new label;
- institutional consequences disappearing;
- supporting cast becoming static;
- villains becoming cartoonishly anti-AI or pro-AI;
- Book 1 trauma being referenced without evolving;
- lack of escalation in the long arc;
- overstuffing canon with trivia.
